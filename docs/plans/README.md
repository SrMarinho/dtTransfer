## Como usar

- Para visão geral e priorização do que deve vir a seguir, leia [`roadmap.md`](roadmap.md). Ele lista todas as melhorias propostas em ordem decrescente de impacto.
- Quando um item do roadmap for aprovado para execução, ele ganha seu próprio arquivo aqui, no formato de `sla-implementation.md` (que serve de modelo).
- Status sugeridos: `📍 rascunho` → `🟡 em discussão` → `✅ aprovado` → `🚀 em execução` → `✔️ concluído`.
